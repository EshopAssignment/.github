# E-Commerce Student Project

This project is a proof-of-concept e-commerce solution developed as part of a LIA internship assignment. The goal is to demonstrate a working online store with the most important parts of a modern shopping flow, from product browsing to checkout and order handling.

## Project Overview

The application is based on a fictional customer case where the customer wants to evaluate an e-commerce platform before deciding whether to build a full-scale solution. The project focuses on building a practical foundation for an online shop while also introducing production-oriented concepts such as performance, testing, containerization, and maintainable architecture.

## Main Features

- Product catalogue with product listing and product detail pages
- Product categories, filtering, sorting, search, and pagination
- Shopping cart with cart rows, pricing, shipping cost, and totals
- Checkout flow with delivery address, Klarna checkout, and PostNord integration
- Order confirmation and customer order history
- Administration area for products, orders, users, and customers
- User login, customer pages, and role-based handling
- System logging and order confirmation email support

## Continued Development Focus

The second part of the assignment expands the project with a stronger focus on making the solution more production-ready. This includes caching to improve performance, event handling for a more loosely coupled architecture, Docker support for consistent development and testing environments, and automated tests for key application behavior.

Additional CI/CD work may be added to automate builds and tests when changes are pushed to the repository.

## Purpose

The project is intended as a learning-focused e-commerce implementation where core webshop functionality is combined with real-world development practices. It gives room to work with integrations, architecture, testing, deployment preparation, and the practical decisions involved in building a scalable web application.
